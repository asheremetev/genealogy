## 📌 Общие принципы

1. **Одна сущность — один файл** (Single Responsibility).
2. **Связи через ссылки**, не дублирование данных (DRY).
3. **Один источник правды** — данные в YAML, отображение через Dataview.
4. **Сначала персона, потом связи** — создай всех участников, потом свяжи.
5. **Минимум → детали** — можно создать запись с минимумом полей, дополнить позже.
6. **Стабильные идентификаторы** — имя файла не должно меняться при изменении данных.

## 🔗 Как правильно ссылаться на людей

> Для женщин — **фамилия при рождении**
> Для мужчин — обычная фамилия (редко меняется)

```yaml
# ✅ Правильно
мать: '[[Петрова Анна Ивановна (1890)]]'
жена: '[[Петрова Анна Ивановна (1890)]]'

# ❌ Неправильно (фамилия по мужу)
мать: '[[Иванова Анна Ивановна (1890)]]'
```

## 📋 Чек-лист полной записи персоны

**Минимум для создания** (обязательные поля схемы):

- [ ] `имя` заполнено
- [ ] `пол` указан
- [ ] `поколение` установлено
- [ ] `достоверность` оценена

**Для полноты** (работа на будущее):

- [ ] `фамилия` (и `фамилия_при_рождении` для женщин)
- [ ] Дата рождения (хотя бы год)
- [ ] Место рождения → ссылка на Place
- [ ] Отец → ссылка (обычная фамилия)
- [ ] Мать → ссылка (**фамилия при рождении!**)
- [ ] Хотя бы один источник привязан
- [ ] `статус_исследования` актуален

# 👤 Персоны

## 🧑 Сценарий: Добавить нового человека

### Шаги

1. Создай файл в `persons/` с именем:
    - **Мужчина:** `Фамилия Имя Отчество (год_рождения).md`
    - **Женщина:** `Фамилия_при_рождении Имя Отчество (год_рождения).md`
    - Если год приблизительный: `... (~1850).md`
    - Если год неизвестен: без года
2. Шаблон применится автоматически (через Templater).
3. Заполни **обязательные** поля: `имя`, `пол`, `поколение`, `достоверность`.
4. Для женщин заполни `фамилия_при_рождении`.
5. Укажи `отец` и `мать` как `[[ссылки]]` (для матери — фамилия при рождении!).
6. Если родители не созданы — сначала создай их (пусть с минимумом данных).

### Пример: мужчина

```yaml
тип: персона
фамилия: 'Иванов'
имя: 'Пётр'
отчество: 'Сергеевич'
пол: М
дата_рождения: 1887-03-15
место_рождения: '[[г. Кострома]]'
отец: '[[Иванов Сергей Николаевич (1861)]]'
мать: '[[Комарова Мария Ивановна (1865)]]' # Фамилия при рождении!
поколение: 2
достоверность: высокая
```

### Пример: женщина

```yaml
# Файл: persons/Петрова Анна Ивановна (1890).md

тип: персона
фамилия: 'Иванова' # Текущая (по мужу)
фамилия_при_рождении: 'Петрова' # При рождении
имя: 'Анна'
отчество: 'Ивановна'
пол: Ж
дата_рождения: 1890-05-12
отец: '[[Петров Иван Сергеевич (1862)]]'
мать: '[[Сидорова Мария Фёдоровна (1865)]]'
поколение: 2
достоверность: высокая
```

## 🏺 Сценарий: Древний предок без фамилии

### Когда это бывает

До ~XVIII века у крестьян и части горожан фамилий не было. В документах:

- Ревизские сказки: `"Иванъ Петровъ сынъ"`
- Метрики: `"крестьянин Андрей Иванов"` (Иванов = сын Ивана, не фамилия)
- Исповедные росписи: просто `"Якимъ"` или `"Арина"`

### Правила именования файлов

```
Имя Отчество (год).md          # Имя + отчество
Имя Отчество (~1650).md        # Приблизительный год
Имя Отчество.md                # Год неизвестен совсем
Имя (год).md                   # Отчество тоже неизвестно
```

### Заполнение полей

```yaml
тип: персона
фамилия: '' # Оставить пустым — фамилии нет
фамилия_при_рождении: '' # Оставить пустым
имя_в_источнике: 'Иванъ Петровъ сынъ' # Дословно из документа!
прозвище: 'Рыжий' # Если есть неофициальное прозвище
имя: 'Иван' # Нормализованное современное написание
отчество: 'Петрович' # Нормализованное
достоверность: предположение
```

### Как ссылаться на таких людей

```yaml
# В записи ребёнка:
отец: '[[Иван Андреевич (1735)]]'
мать: '[[Федора Леонтьевна (1732)]]'

# В семье — полное имя в названии файла:
families/Иван Андреевич - Федора Леонтьевна (~1759).md
```

### Важно

> Поле `имя_в_источнике` заполняй **всегда**, когда есть документ. Даже если написание совпадает с нормализованным — это подтверждает, что данные из источника, а не выдуманы.
>
> Поле `прозвище` заполняй, если прозвище встречается в документах или передавалось устно. Иногда прозвища стали фамилиями потомков — это важная генеалогическая связь.

## 🔍 Сценарий: Неизвестная фамилия при рождении

### Если знаем только фамилию по мужу

```yaml
# Файл: persons/Иванова Мария Петровна (~1860).md
# (Известна только как "вдова Иванова" из переписи)

фамилия: 'Иванова'
фамилия_при_рождении: '' # Неизвестна
достоверность: низкая
```

### Если потом найдёшь фамилию при рождении

1. Переименуй файл: `Иванова...` → `Сидорова Мария Петровна (~1860).md`
2. Obsidian предложит обновить все ссылки — согласись.
3. Заполни `фамилия_при_рождении: "Сидорова"`.

## 👨 Сценарий: Мужчина сменил фамилию

Редкий случай (усыновление, псевдоним).

### Подход тот же, что у женщин

- **Имя файла** = фамилия при рождении
- **`фамилия`** = текущая/известная
- **`фамилия_при_рождении`** = исходная

### Пример

```yaml
# Файл: persons/Джугашвили Иосиф Виссарионович (1878).md

фамилия: 'Сталин'
фамилия_при_рождении: 'Джугашвили'
```

# 👨‍👩‍👧 Семьи

## 👰 Сценарий: Женщина вышла замуж (смена фамилии)

### Золотое правило

> **Имя файла = фамилия при рождении**
> **Имя семьи = ... - фамилия жены при рождении**

### Почему?

| Причина                 | Объяснение                                  |
| ----------------------- | ------------------------------------------- |
| **Стабильность ссылок** | Файл не переименовывается при браке/разводе |
| **Связь с родителями**  | Фамилия совпадает с фамилией отца           |
| **Архивный поиск**      | В метриках до брака — девичья фамилия       |
| **Множественные браки** | Нет путаницы с фамилиями                    |

### Шаги

1. Создай файл с фамилией при рождении:
    ```
    persons/Петрова Анна Ивановна (1890).md
    ```
2. Заполни оба поля фамилии:
    ```yaml
    фамилия: 'Иванова' # Текущая (по мужу)
    фамилия_при_рождении: 'Петрова' # При рождении
    ```
3. Ссылки на эту персону всегда используют фамилию при рождении:

    ```yaml
    # В записи мужа:
    жена: '[[Петрова Анна Ивановна (1890)]]'

    # В записи ребёнка:
    мать: '[[Петрова Анна Ивановна (1890)]]'
    ```

## 💒 Сценарий: Создание семьи (брак)

### Шаги

1. Убедись, что оба супруга существуют в `persons/`.
2. Создай файл в `families/`:
    ```
    Полное_имя_мужа - Полное_имя_жены_при_рождении (год_брака).md
    ```
    Полное имя = **Фамилия Имя Отчество**. Если фамилии нет — **Имя Отчество**.
3. Заполни `муж`, `жена`, `дата_брака`, `место_брака`.
4. Установи `статус_брака: в_браке`.

> **Важно:** В имени семьи всегда фамилия жены **при рождении**, даже если она уже была замужем!

### Правило именования

Всегда используй полное имя (всё, что есть). Если фамилии нет — Имя Отчество:

```
# Оба с фамилиями
Иванов Пётр Сергеевич - Петрова Анна Ивановна (1912).md

# У жены нет фамилии
Шереметьев Игнатий Кузьмич - Елена Ивановна (~1778).md

# У мужа нет фамилии
Ермилий Трофимович - Шереметьева Федосья Игнатьева (1805).md

# Ни у кого нет фамилии
Иван Андреевич - Федора Леонтьевна (~1759).md
```

### Пример

```yaml
# Файл: families/Иванов Пётр Сергеевич - Петрова Анна Ивановна (1912).md

тип: семья
муж: '[[Иванов Пётр Сергеевич (1887)]]'
жена: '[[Петрова Анна Ивановна (1890)]]' # Фамилия при рождении!
дата_брака: 1912-05-20
место_брака: '[[г. Кострома]]'
место_венчания: '[[г. Кострома]]'
статус_брака: в_браке
достоверность: высокая
```

## 💒💒 Сценарий: Повторный брак (2-я, 3-я жена/муж)

### Принцип

> В имени **каждой** семьи — фамилия жены **при рождении** (одна и та же!)

### Шаги

1. **Обнови первую семью** — измени `статус_брака`:
    - Супруг(а) умер(ла): `вдовство`
    - Развод: `развод` + `дата_развода`
2. Создай новый файл семьи (полные ФИО обоих):
    ```
    families/Сидоров Иван Петрович - Петрова Анна Ивановна (1920).md
    ```
    (Та же Петрова, другой муж.)
3. У детей от нового брака укажи обоих родителей.

### Пример: три брака одной женщины

```
Файл персоны: persons/Петрова Анна Ивановна (1890).md
  фамилия: "Козлова"              # Текущая (по 3-му мужу)
  фамилия_при_рождении: "Петрова" # Неизменная

Файлы семей:
  families/Иванов Пётр Сергеевич - Петрова Анна Ивановна (1912).md  # 1-й брак
    статус_брака: вдовство

  families/Сидоров Иван Петрович - Петрова Анна Ивановна (1920).md  # 2-й брак
    статус_брака: развод
    дата_развода: 1925

  families/Козлов Фёдор Андреевич - Петрова Анна Ивановна (1926).md # 3-й брак
    статус_брака: в_браке
```

**Во всех трёх семьях ссылка на жену одинаковая:**

```yaml
жена: '[[Петрова Анна Ивановна (1890)]]'
```

### Если мужчина женился несколько раз

Аналогично — каждый брак в отдельном файле, в имени — фамилия соответствующей жены при рождении:

```
families/Иванов Пётр Сергеевич - Петрова Анна Ивановна (1912).md   # 1-я жена
families/Иванов Пётр Сергеевич - Сидорова Мария Фёдоровна (1925).md # 2-я жена
families/Иванов Пётр Сергеевич - Козлова Елена Павловна (1930).md   # 3-я жена
```

## 💔 Сценарий: Развод и смена фамилии

### Что делать

| Ситуация                     | Действие                           |
| ---------------------------- | ---------------------------------- |
| Оставила фамилию мужа        | `фамилия` не меняется              |
| Вернула фамилию при рождении | `фамилия` = `фамилия_при_рождении` |
| Взяла фамилию нового мужа    | `фамилия` = новая фамилия          |

### Шаги

1. Обнови семью:
    ```yaml
    статус_брака: развод
    дата_развода: 1925-06-15
    ```
2. Обнови персону (если фамилия изменилась):
    ```yaml
    # Вернула девичью:
    фамилия: 'Петрова'
    фамилия_при_рождении: 'Петрова'
    ```
3. **Имя файла НЕ меняется** — `Петрова Анна Ивановна (1890).md`.

## 👶 Сценарий: Родился ребёнок

### Шаги

1. Создай файл ребёнка в `persons/`.
2. Заполни `отец` и `мать`:
    ```yaml
    отец: '[[Иванов Пётр Сергеевич (1887)]]'
    мать: '[[Петрова Анна Ивановна (1890)]]' # Фамилия при рождении!
    ```
3. Установи `поколение` (на 1 меньше родителей).
4. Ребёнок автоматически появится:
    - В карточке обоих родителей.
    - В карточке семьи родителей.
    - В карточках братьев/сестёр.

# 📍 Места

## 📍 Сценарий: Одно место связано с многими людьми

### Это нормально!

Место — это отдельная сущность, на которую ссылаются многие записи:

```
places/г. Санкт-Петербург.md
  ↑
  ├── Иванов Пётр Сергеевич (1887).md                        → место_рождения
  ├── Петрова Анна Ивановна (1890).md                        → место_смерти
  ├── Сидоров Иван Петрович (1920).md                        → место_рождения
  └── Иванов Пётр Сергеевич - Петрова Анна Ивановна (1912).md → место_брака
```

### Преимущества

- **Нет дублирования** — координаты и история в одном месте.
- **Автоматические связи** — Dataview покажет всех связанных.
- **Единообразие** — одинаковое написание везде.

---

## 📍 Сценарий: Одноимённые населённые пункты

Если есть два места с одинаковым названием в разных регионах — уточняй в скобках:

```
places/с. Никольское (Костромская губ).md
places/с. Никольское (Саратовская губ).md
```

Уточнение — в конце, после пробела. Скобки — круглые. Внутри — минимально достаточный признак (губерния, уезд, район).

Формат имени по типам мест смотри в [[data-schema#именование-мест|Схеме данных]].

# 📄 Источники, события, истории, исследования

## 📚 Сценарий: Работа с архивной записью

### Шаги

1. Создай источник в `sources/`:
    ```
    Метрическая книга - Кострома (1887).md
    ```
2. Заполни реквизиты: `архив`, `фонд`, `опись`, `дело`, `лист`.
3. В `персоны` укажи всех упоминаемых (фамилии при рождении для женщин!).
4. Добавь скан: `![[media/scans/mk-kostroma-1887-l42.jpg]]`.
5. Заполни «Извлечённые факты» в теле заметки.
6. Обнови данные в карточках персон.

---

## 🔗 Сценарий: Привязать источник к нескольким персонам

Один документ часто содержит сразу несколько человек (метрика о браке — жених, невеста, свидетели; перепись — вся семья).

В источнике перечисляй всех:

```yaml
# sources/Перепись 1897 - д. Горбово.md

тип: источник
категория: перепись
год_документа: 1897
персоны:
    - '[[Балясников Иван Петрович (1880)]]'
    - '[[Титова Анна Ефимовна (1883)]]' # Фамилия при рождении!
    - '[[Балясникова Мария Ивановна (1907)]]'
    - '[[Балясников Дмитрий Иванович (1911)]]'
```

Обратная связь выстраивается автоматически — в карточке каждой персоны появится источник в разделе «Источники».

---

## 📅 Сценарий: Добавить событие

События — для фактов, которые не ложатся в «рождение/брак/смерть»: миграция, мобилизация, арест, раскулачивание, эмиграция.

### Шаги

1. Создай файл в `events/` с понятным названием:
    ```
    events/Мобилизация 1914 - Иванов Пётр.md
    ```
2. Заполни `вид_события`, `дата`, `место`, `участники`.
3. Привяжи источники (если есть).

### Пример

```yaml
тип: событие
вид_события: мобилизация
дата: 1914-08-01
место: '[[г. Кострома]]'
участники:
    - '[[Иванов Пётр Сергеевич (1887)]]'

    - '[[Воинский призыв 1914]]'

```

Длящиеся события (ссылка, раскулачивание, эвакуация) — указывай `дата_окончания`.

## 📖 Сценарий: Записать устную историю

Устные рассказы, воспоминания, дневники, письма.

### Шаги

1. Создай файл в `stories/` — короткий описательный заголовок.
2. Укажи `рассказчик` (кто рассказал).
3. Перечисли упомянутых `персоны` и `места`.
4. В теле — **запиши максимально близко к оригиналу**. Отдельно — своё мнение в блоке «Комментарии исследователя».

### Пример

```yaml
тип: история
заголовок: 'Как дед попал на Хабаровск'
рассказчик: '[[Шереметьев Анатолий Викторович (1931)]]'
дата_записи: 2024-05-10
период_событий: '1940-1945'
персоны:
    - '[[Шереметьев Виктор Васильевич (1909)]]'
места:
    - '[[г. Хабаровск]]'
источник_истории: устный_рассказ
достоверность: средняя
```

## 🎯 Сценарий: Сессия исследования

Журнал исследования — заметки о том, где искал, что нашёл и куда идти дальше. Помогает не искать одно и то же дважды.

### Шаги

1. Создай файл в `reports/` через шаблон `research-entry.md`:
    ```
    reports/2026-04-22 - метрики Псковская губерния.md
    ```
2. В YAML: `цель`, `объекты` (на кого/что направлено исследование).
3. В теле — таблица «Где искал», список «Что нашёл», «Следующий шаг».
4. По итогам — обнови карточки персон / создай новые источники.

### Пример

```yaml
тип: исследование
дата: 2026-04-22
цель: 'Найти метрики о рождении детей Шостаковича В.М.'
объекты:
    - '[[Шостакович Винцент Михайлович (~1803)]]'
результат: 'Найдены 2 из 4 ожидаемых рождений'
```

Журнал сессий собирается в [[research-log]].
