# 📐 Схема данных — Генеалогический Vault

Единый справочник всех YAML-полей. При сомнениях — сверяйся с этим файлом.

---

## Общие правила

### Форматы дат

| Точность       | Формат          | Пример       |
| -------------- | --------------- | ------------ |
| Полная         | `YYYY-MM-DD`    | `1887-03-15` |
| Месяц          | `YYYY-MM`       | `1887-03`    |
| Год            | `YYYY`          | `1887`       |
| Приблизительно | `~YYYY`         | `~1887`      |
| Неизвестно     | оставить пустым |              |

### Ссылки

- Всегда `[[Имя файла]]` без пути к папке
- Имя в ссылке = точное имя файла (с годом в скобках)
- Для списков: `["[[Файл 1]]", "[[Файл 2]]"]`
- Для женщин: всегда используй **фамилию при рождении** в ссылках

### Допустимые значения (enum)

Валидатор [data-quality](_dashboards/data-quality.md) покажет ошибки при несоответствии.

| `место_брака`    | ссылка |              | `[[Place]]`                                        |
| `место_венчания` | ссылка |              | `[[Place]]`                                        |
| `дата_развода`   | дата   |              |                                                    |
| `статус_брака`   | строка |              | `в_браке` · `вдовство` · `развод` · `неизвестно`   |
| `достоверность`  | строка |              | `высокая` · `средняя` · `низкая` · `предположение` |
| `теги`           | список |              | `[семья]`                                          |
| `создано`        | дата   |      ✅       |                                                    |
| `обновлено`      | дата   |      ✅       |                                                    |

> **Правило именования файлов семей:**
>
> - Формат: `Фамилия_мужа - Фамилия_жены_при_рождении (год_брака).md`
> - Пример: `Иванов - Петрова (1912).md` (даже если жена уже была Сидорова по первому мужу)

> Дети **не перечисляются** в семье — они определяются автоматически через `отец`/`мать` в записях детей.
